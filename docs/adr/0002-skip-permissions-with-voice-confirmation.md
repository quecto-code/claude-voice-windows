# パーミッションは常時バイパスし、破壊的操作のみ音声で確認する

音声モードの Claude Code は `--dangerously-skip-permissions` で起動し、ハーネス標準のキーボード承認プロンプトを一切出さない。その代償として、Claude が破壊的・不可逆な操作を行う前に、その内容を音声で伝えて実行可否を尋ね（requirements の A-2 / voice_speak の「許可要求」役割）、`voice_listen` で得た音声の「はい」でのみ実行する。

## なぜ

ハンズフリーが本機能の中核目的であり、キーボード承認プロンプトが出た時点でその場で手を使わざるを得ず、目的が破綻する。プロンプトを出させないにはパーミッションをバイパスするしかない。安全性は、ハーネスではなく Claude 自身の事前音声確認に移し替える。これは [ADR-0001](./0001-mcp-tool-driven-turn-based.md) のターン制（処理中に止められない代わりに事前確認する）の安全ストーリーと整合する。

> **更新 (2026-05-31):** 委譲型準並行（[ADR-0004](./0004-delegated-quasi-concurrency.md)）の導入後も、本 ADR の安全不変条件（破壊的操作は事前に音声でブロッキング確認、はいでのみ実行）は緩めない。準並行下では走行中ジョブを音声で即時 kill できないため、**破壊的操作はフォアグラウンドでのみ実行し、バックグラウンドジョブは破壊的操作を親に差し戻して音声確認を経る**。準並行下の安全ストーリーの詳細は ADR-0004 を参照。

## Consequences

- 安全性の担保がハーネスから Claude の判断（破壊的操作の前に許可要求を出すか）に依存する。確認対象は破壊的・不可逆な操作に限定し、通常操作は宣言（事前報告）のみで進める。
- 確認すべき操作の線引きと粒度は、使いながら調整する前提（固定仕様ではない）。
- 委譲型準並行（ADR-0004）下では、破壊的操作はフォアグラウンドに限定する（委譲先は実行せず親に差し戻す）。
