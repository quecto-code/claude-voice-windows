# 音声ループは MCP ツール駆動のターン制とする

> **更新 (2026-05-31):** 本 ADR のうち「ターン制（処理中は割り込み不可）」の部分は [ADR-0004](./0004-delegated-quasi-concurrency.md)（委譲型準並行）に置き換わった。以下の本文は決定当時の記録として保持する（MCP ツール駆動・サーバステートレス・ループは `/voice-chat` 側、という骨格は ADR-0004 でも引き続き有効）。

claude-voice は、Claude 自身が `listen` / `speak` 系の MCP ツールを順に呼んでループを回す「MCP ツール駆動」方式を採る。Claude は 1 ターン内で逐次実行されるため、処理中に同時に音声を聞くことは原理的にできず、対話モデルはターン制（処理中は割り込み不可、requirements のモデル A）に確定する。

## Considered Options

- **(a) MCP ツール駆動 / ターン制（採用）**: requirements の MCP 前提に素直に乗り、実装が単純。個人用・週N回利用の規模では十分実用。
- **(b) 外部プロセス駆動 / 割り込み可能（モデル B）**: 別プロセスでマイクを常時受け、Claude のターンを kill/ESC で中断する。処理中の割り込みが可能になる反面、Claude Code の起動・制御を外側から握る必要があり実装が重い。requirements 自身が「実行中の処理を実際に中断できるか」を要検証としておりリスクが高いため見送り。

## Consequences

- requirements の「対話モデル」関連要件のうち、モデル A（A-1, A-2）を採用し、モデル B（B-1〜B-4）は対象外となる。
- 「料理中に進行中の処理へ割り込む」体験は得られない。割り込みたい場合はターンの完了を待つ必要がある。
