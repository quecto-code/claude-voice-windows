# claude-voice

Claude Code をハンズフリーで使うための音声入出力インターフェース。ユーザの音声指示を文字起こしして Claude Code に渡し、Claude の応答や処理完了を音声で読み上げる。MCP ツール駆動で動き、重い作業はバックグラウンドに委譲する**委譲型準並行**（[ADR-0004](./adr/0004-delegated-quasi-concurrency.md)。当初の[ADR-0001](./adr/0001-mcp-tool-driven-turn-based.md) のターン制を更新）。

## Language

**音声モード**:
ユーザが音声で指示を出し、応答を音声で受け取れる対話状態の全体。開始から終了コマンドまでの間、ターンを繰り返すループとして動く。手動で開始する（ホットワード起動はしない）。

**ターン**:
フォアグラウンドの「listen 状態 → 軽い処理または委譲 → 読み上げ」の 1 サイクル。重い作業は**バックグラウンドジョブ**に委譲され、ターンと並行して走る（[ADR-0004](./adr/0004-delegated-quasi-concurrency.md)）。

**委譲 / バックグラウンドジョブ**:
重い作業（数十秒以上かかる検索・複数ファイル編集・テスト・ビルド等）を、フォアグラウンドがバックグラウンド・サブエージェントに渡して並行実行させること（委譲）、およびその実行単位（バックグラウンドジョブ）。同時に走るのは 1 個まで。ジョブは音声状態（silent カウント等）に触れず、結果・進捗はテキストで親（フォアグラウンド）に返す。
_Avoid_: 「並列」（同時複数を連想させる。当面は単一ジョブのため使わない）

**委譲型準並行**:
会話を止めずに重い作業を裏で進める対話モデル。走行中ジョブの即時 kill（完全な割り込み）は狙わないため「準」並行であり、完了・進捗の読み上げはターンの切れ目で行う。requirements の「モデル A（ターン制）／モデル B（割り込み可）」の二択ではなく、第 3 のモデルとして採る（[ADR-0004](./adr/0004-delegated-quasi-concurrency.md)）。

**listen 状態**:
システムがユーザの音声入力を受け付けている状態。この状態に入ったことは音声でユーザに伝える。

**確定**:
listen 状態で受けた発話を「1 つの指示」として閉じ、処理に渡すこと。確定の引き金は 2 つ — 一定時間の無音（沈黙確定）と、ユーザが発する確定ワード。
_Avoid_: 「終了」（音声モードの終了と紛らわしいため、発話を閉じる意味では使わない）

**確定ワード**:
ユーザが発すると、その時点までの発話を即座に確定させる特定の語。沈黙を待たずに処理を開始させるための合図。
_Avoid_: 終了ワード

**終了コマンド**:
音声モード全体を終わらせる指示。listen 状態に戻らずループを抜ける。確定（発話を閉じる）とは別概念。

**読み上げの 4 役割**:
voice_speak が音声に出す内容は次の 4 つに限る — **事前宣言**（「～をする」これからの処理の要約）、**進捗報告**（走行中のバックグラウンドジョブの「まだ～中」）、**事後報告**（「～をした」処理結果の要約）、**許可要求**（破壊的操作の実行可否を問う）。このうちブロッキングで返事を待つのは許可要求のみ。音声を出すのはフォアグラウンドのみ（**単一スピーカー**）— バックグラウンドジョブは voice ツールを呼ばず、結果・進捗はテキストで親に返す（[ADR-0004](./adr/0004-delegated-quasi-concurrency.md)）。

## 解決済みの用語整理

- **「終了」の二義性（解決済み）**: requirements では Requirement 9 の「終了ワード」（発話を閉じる）と Requirement 7 の「音声モードの終了」（モードを抜ける）に同じ「終了」が使われていた。前者を **確定ワード**、後者を **終了コマンド** と分けて呼ぶことで確定。requirements 上の「終了ワード」は本 glossary の「確定ワード」を指す。
- **「ターン」の排他性の撤回（解決済み）**: 当初のターンは「処理と listen 状態は排他的、処理中は受け付けない」と定義していた（ADR-0001 のターン制由来）。委譲型準並行（[ADR-0004](./adr/0004-delegated-quasi-concurrency.md)）の採用に伴い、重い作業はバックグラウンドジョブに委譲してターンと**並行**に走るため、この排他性を撤回した。ただしフォアグラウンドのターン自体は依然逐次で、完了・進捗の読み上げはターンの切れ目で行う（＝完全並行ではなく**準**並行）。

## Example dialogue

> 開発者: 音声モードを開始すると、まず何が起きる？
> 作者: listen 状態に入って「聞いてます」と読み上げる。そこで喋ると、黙れば沈黙確定、急ぎたければ確定ワードを言えばその場で確定して処理に回る。
> 開発者: じゃあ「終了」って言ったら？
> 作者: それは終了コマンド。発話を閉じる確定ワードとは別物で、音声モードそのものを抜ける。
